How to Use:

  1. Place a switch of your choosing in the editor and double-click it to open up it's settings.
  2. Customize colors if desired.
  3. Under "Name of Lights to Control", type in a name of your choosing. I recommend typing a name that describes the type or location of the switch to keep things organized (ie. "keySwitch" or "backDoor").
    • If you want the switch to be passive and not control any lights, simply leave "Name of Lights to Control" empty.
  4. Then, for all the lights you want the switch to control, double-click them and change their name to the name you chose in step 3.
The following additional switch settings can also be set:
  • Start "On": Will make the switch (and connected lights) start in the on position when the simulation starts.
  • Flicker Time (Seconds): Amount of time lights will flicker / warm up after being turned on. Set to 0 to disable.
  • Override Interval (Seconds): How often to set the current state of the switch to all lights. "0" disables. Attempts to keep lights with their own on/off scripts "synced up" to the switch. Values close to 0 may cause lag.
  • Switch On/Off Sound Effects: Sound effect file to play when the switch is turned on or off.
  • Disable Entity Color Changing: Disables changing the alpha channel of entity slot 0 (used to make vanilla lights appear fully "off"). Useful if colors or textures are being negatively affected on 3rd party lights.

Compatibility:

  • All vanilla NL2 scenery object lights.
  • Any un-scripted 3rd party custom scenery objects (SCOs) with any amount of lights.
  • Most scripted 3rd party lights (ie. animated lights, track lights, etc.) can be made to work with the right settings, but results will vary. The switch's "Override Interval" and "Disable Entity Color Changing" settings can be used in effort to increase compatibility.
  • Coaster running lights unfortunately cannot be controlled due to NL2 limitations.

Known Issues:

  • Different lights within a single SCO unfortunately cannot be controlled separately by two different switches at this time. Use two separate SCOs to work around this.
  • If an override interval is set, multiple switches cannot be "wired in series" with each other to control the same group of lights (flickering will occur). If the interval is disabled, multiple switches can control the same group of lights, but the switches' current state will temporarily de-sync until it is flipped again.
  • Flicker should be disabled when controlling 3rd party lights that change color over time. Otherwise, the switch may change the light(s) to an undesired color.
